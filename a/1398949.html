用户属性

📄 1,620 字 · 🕐 2024-12-12

属性用于描述事件发生时的各维度信息，包括用户信息（user_id）、设备信息（app版本、设备号、ip地址、设备型号等）、业务自定义信息（上报到params的自定义参数）等。属性通常包括事件属性和用户属性，本文为您介绍用户属性的操作要点。

<span id="ce74d14f"></span>

基本概念

术语概念介绍
用户事件行为的发生主体人，当前支持使用user_id、ssid、device_id来唯一标识一个用户，详情请参见支持的用户唯一标识。
用户属性* 和行为无关的用户自身状态的属性，例如年龄、性别等。<br><br>* 和行为有关，但不变或者不经常变化的属性，例如注册时间、VIP等级、最近一次支付时间等。
预置用户属性预置用户属性：SDK预置的用户属性。<br><br>不同数据上报端支持的预置事件列表不同，预置事件列表详情请参见预置属性总表。

<span id="f2493edb"></span>

功能介绍

<span id="93f664cb"></span>

新增用户属性

如果您需要单独登记用户属性，可执行以下操作：

  1. 选择数据管理 \> 元数据管理 \> 用户属性，进入用户属性页面。
  1. 单击新增用户属性按钮，弹出新增用户属性侧拉窗口，设置相关信息后，单击确定按钮完成新增用户属性。

<span> </span>

参数配置说明
属性名、展示名* 用户属性命名时要具体，谨慎使用泛化的参数名称。<br><br>* 如果业务含义相同，尽量复用已经定义的字段，避免随意定义后导致系统中字段泛滥。<br><br>* DataTester SDK 已经预置了常见的用户属性，无需重复登记。预置事件列表详情请参见预置属性总表。
数据类型在下拉框中选择用户属性的数据类型，当前支持int、string、float、datetime、list、version类型。<br><br><br>* 选择的用户属性类型需与后续上报的用户属性的数据类型一致，否则可能导致数据无法正常落库。<br><br>* 上报的数据类型与落库的数据类型对应关系及详细的数据格式要求请参见支持的数据格式（自定义事件/属性）。
计算逻辑当前支持按需选择计算全部值或计算最终值。<br><br><br>* 计算全部值：分析该属性变化过程中出现的所有取值，示例：属性为“会员类别”，用户a在某天从“普通会员”升级到“VIP”，“VIP”是最新取值，则用户a发生的访问、购买等事件改变前会被归到“普通用户”上，改变当天和改变后会被归到“VIP”上。<br><br>* 计算最终值：仅分析该属性变化的最终取值。示例：例如：属性为“会员类别”，用户a在某天从“普通会员”升级到“VIP”，“VIP”是最新取值，则用户a历史上发生的访问、购买等所有事件都会归到“VIP”上。
属性描述自定义属性的描述，便于标识用户属性。

<span id="cae131ad"></span>

验收属性

用户属性列表仅展示已验收/登记的用户属性，未验收属性可点击“验收属性”按钮，在“数据验收”页面查看并完成验收。

您也可以在“数据验收”页面管理用户属性黑名单，将需要的用户属性添加或移出黑名单。

<span> </span>

<span id="2bbdc7ce"></span>

修改属性数据类型

如果您此前登记的数据类型有误或变更了新的类型，可以在列表中修改数据类型字段。

<div data-tips="true" data-tips-type="warning" data-tips-is-title="true">注意</div>

<div data-tips="true" data-tips-type="warning">变更类型分兼容变更和非兼容变更， 大类型向小类型转不丢历史数据(比如 string \-\> int)，小类型往大类型转会丢历史数据(比如 int \-\> float)。请谨慎选择数据类型。</div>

<span> </span>

<span id="04e8f4bf"></span>

批量编辑展示名、属性描述

如果您希望修改用户属性的展示名或属性，您可以：

<span> </span>

  • 单个修改：直接单击某个属性后的编辑或添加按钮，进行修改用户属性的展示名或描述。
  • 批量修改：选择多个待修改的用户属性后，单击底部的批量导出按钮，导出后在本地修改完成后，再单击顶部的批量修改按钮，将修改结果批量导入。
