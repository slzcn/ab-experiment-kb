3. 接口描述

<span id="_3-1-newclient"></span>

3.1 NewClient

接口：

NewClient(token string, configs ...config.Func) \*AbClient

描述： 初始化ABTest分流

参数：

  • token：表明您的Tester应用。出于安全考虑，此处使用的token=appKey，而非appId。

在集团中接入一个应用后，您可以在集团相关页面查看应用的AppKey等信息，详情请参考：如何创建应用。

  • SaaS\-云原生

<span> </span>

  • SaaS\-非云原生

<span> </span>

<div data-tips="true" data-tips-type="tip" data-tips-is-title="true" data-wrapper-indent="1">说明 </div>

<div data-tips="true" data-tips-type="tip" data-wrapper-indent="1">请尽早初始化AbClient，以免影响您的分流服务和埋点上报服务。 </div>

  • config.WithMetaHost：获取元信息的地址，默认为SaaS\-非云原生国内地址；海外Saas域名：config.MetaHostSG；SaaS\-云原生/私有化需要设置为产品域名
  • config.WithTrackHost：事件上报的地址，默认为SaaS\-非云原生国内地址；海外Saas域名：config.TrackHostSG；SaaS\-云原生/私有化需要设置为上报域名
  • config.WithFetchInterval(60 \* time.Second)：meta更新间隔，默认为60s，一般不需要设置
  • config.WithWorkerNumOnce(20)：事件上报协程数，一般不需要设置
  • config.WithAnonymousConfig(true, true)：匿名上报配置，第一个参数为开启关闭，第二个参数区分saas和私有化；不使用匿名上报的场景下不需要设置
  • config.WithLogger(log.NewLogrusAdapt(logrus.New()))：自定义日志接口，提供默认实现

<span id="_3-3-activate"></span>

3.3 Activate

接口： func (t \*AbClient) Activate(variantKey, decisionId, trackId string, defaultValue interface{}, attributes map[string]interface{}) (interface{}, error)

描述： 获取特定key的分流结果，并上报曝光事件。

<div data-tips="true" data-tips-type="tip" data-tips-is-title="true">说明</div>

<div data-tips="true" data-tips-type="tip">批量获取分流结果，并上报曝光事件可使用GetExperimentConfigsWithImpressionWithVariantKeys接口，详情请参见下文对应接口的文档。</div>

参数：

variantKey：变体的key

decisionId：本地分流用户标识

trackId：事件上报用户标识

defaultValue：变体默认值

attributes：用户属性

返回值： 该函数返回命中变体对象，未命中时返回默认值对象

<div data-tips="true" data-tips-type="tip" data-tips-is-title="true">说明</div>

<div data-tips="true" data-tips-type="tip">1、该接口与所有含有“WithImpression”字样的接口均会自动上报曝光事件，用户可在AbClient初始化时指定domain，调整事件上报地址。</div>

<div data-tips="true" data-tips-type="tip">2、请务必填写trackId字段，否则上报失效</div>

<span id="_4-1-userabinfohandler"></span>

4.1 UserAbInfoHandler

用户信息处理接口，冻结实验、进组不出组场景下使用

<div data-tips="true" data-tips-type="tip" data-tips-is-title="true">说明</div>

<div data-tips="true" data-tips-type="tip">冻结实验和进组不出组需要持久化用户的进组信息，SDK提供的默认实现不进行数据持久化；</div>

<div data-tips="true" data-tips-type="tip">如有业务需要，则实现UserAbInfoHandler接口，结合Redis或其他外部存储对用户进组信息进行持久化处理，初始化AbClient时传入。</div>

<div data-tips="true" data-tips-type="tip">使用方式：</div>

  1. <div data-tips="true" data-tips-type="tip">使用NewClient初始化AbClient时默认使用空实现，不启用“进组不出组”功能</div>
  1. <div data-tips="true" data-tips-type="tip">继承UserAbInfoHandler接口，自行实现持久化存储；使用NewClientWithUserAbInfo初始化AbClient，并传入自行实现的UserAbInfoHandler类，则可启用“进组不出组”功能</div>

使用Redis缓存示例（仅供参考）

client.NewClientWithUserAbInfo("token", NewRedisUserAbInfoHandler())

func (u *RedisAbInfoHandler) NeedPersistData() bool {

<span id="_4-2-匿名上报"></span>

4.2 匿名上报

获取不到uuid的用户，可以通过填充device_id或者web_id进行事件上报

  1. NewClient时设置匿名上报配置，第一个参数（true/开启，false/关闭）匿名上报，第二个参数（true/saas，false/私有化）
client.NewClient("appKey", config.WithAnonymousConfig(true, true))
  1. 添加device_id, web_id, bddid到用户属性attributes，trackId固定传入空字符串""
trackId = "";
attributes["device_id"] = 1234; int64
attributes["web_id"] = 5678; int64
attributes["bddid"] = "91011"; string
  1. 请求Activate或其他'WithImpression'接口即可匿名上报
