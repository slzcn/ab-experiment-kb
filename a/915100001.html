A/B 测试专家养成之路：知识图谱、能力矩阵与修炼方法

本文整理自内部培训课程《如何成为 A/B 测试专家》，系统梳理成为 A/B 测试专家需要掌握的知识（应知）与锻炼的能力（应会），并给出每一项的获取途径与修炼方法。

做一桌好饭，你需要会用材料（菜、肉、油、料）、掌握程序（炒、煮、煎、炸）、用好工具（锅、碗、瓢、勺）。成为 A/B 测试专家同理，分两条主线：应知（知识图谱）与应会（能力矩阵）。

  • 知识学习：有明确的路径，可按图索骥。
  • 能力训练：没有快速的捷径，需要大量收集、大量刻意练习。

一、知识图谱（应知）：六大领域

A/B 测试专家的知识储备横跨数据、设计、营销三大板块，可细分为六个领域，各约占 10% 权重：

1. 垂直业务领域

作用：理解一切工作的出发点。 想清楚四个问题：

  • 我们的业务在解决用户的什么问题？
  • 我们如何解决，行业又是如何解决的？
  • 线上产品作为解决问题的主要载体，它的定位是什么？
  • 大环境趋势如何，当下要解决的关键问题是什么？

2. 产品设计领域

作用：发现问题、解决问题。 只有广泛学习产品设计原则，才能敏锐发现并准确定义产品问题；结合正确的设计思维，灵活应用设计原则，提出产品与交互设计方案。

推荐书籍：

  • 《About Face 4：交互设计精髓》——数字产品和系统的交互设计指南，全面讲述交互设计的过程、原理与方法，含大量细节指导。
  • 《点石成金：访客至上的 Web 和移动可用性设计秘笈》——剖析用户心理，在使用模式、导航设计、主页布局、可用性测试等方面给出大量简单易行的建议。

进阶渠道：关注优质个人/机构公号、学习大厂用户体验中心的实践。

3. 营销专业领域

作用：洞察客户，了解原因。 要理解购买或转化背后的原因，必须回到"人"的维度，了解客户的考虑、习惯以及环境影响。

  • 购买决策：《消费者行为学——洞察中国消费者》（系统建立本土化消费者决策模型）、《客户为什么会购买》（20 年购物者跟踪研究，剖析购买行为与消费心理的博弈）。
  • 转化决策：《测出转化率——营销优化的科学与艺术》（CRO 领域开山之作）、《超级转化率》（实战手册，大量可直接上手的经验与案例）。

4. 用户数据分析

作用：明确目标，洞察结论。 "如果你不能量化它，你就不能管理它。" 无论测试前做目标规划，还是测试后做分析洞察，都离不开用户体验数据与用户行为数据。

  • 《如何用数据驱动增长》——讲清常见目标、优化对象、优化手段如何用行为数据融合。
  • 《用户体验度量：收集、分析与呈现》——科学度量用户体验的完整方法，帮助厘清"我们要量化的用户体验到底是什么"。

5. 数理统计理论

作用：保证试验结论和决策的科学性。 有限地掌握与 A/B 测试直接相关的数理统计部分，保证试验过程的决策能力和结论解读能力。

  • 《统计学》（贾俊平）——理工科背景可直接看教材，配套做题效果更好。
  • 《深入浅出统计学》——把抽象数学概念生活化、故事化，适合理解概念的现实意涵。
  • 可汗学院公开课·统计学（网易公开课）——涵盖随机变量、均值方差、概率密度、各类分布、大数定律、中心极限定理、参数估计、置信区间、假设检验与 p 值、方差分析、回归分析等。

6. 试验流程规范

作用：规范严谨地试验，以保证结论可信。 只有规范严谨地实施试验，数据才可信；基于可信数据的业务决策才有意义。

  • 《关键迭代：可信赖的线上对照试验》——专业、科学、全面的试验实施经典，针对技术、组织、流程、数学问题给出先进解决方案。
  • 《A/B 测试：创新始于试验》——入门佳作，涵盖试验流程核心工作方法，含大量常见案例，适合业务人员。
  • 极客时间《A/B 测试从 0 到 1》（张博伟）——介于试验专家与业务人员之间的进阶课程，帮助构建完整试验流程体系，并针对常见误区、高频坑点给出解法。

二、相关领域信息渠道（碎片化学习）

三类持续输入的渠道：

  • 营销机构 · CXL（cxl.com）：全球领先的营销教育机构，汇集顶尖营销专家，提供大量 A/B 测试与转化优化的博客文章和学位课程。
  • 营销社区 · HubSpot（hubspot.com）：全球领先的营销系统提供商兼社群，汇集知名营销内容与案例。
  • CRO 研究机构 · WiderFunnel（widerfunnel.com）：CRO 领域研究鼻祖，2009 年提出经久不衰的转化优化范式 LIFT 模型。
  • CRO 研究机构 · Convertize（convertize.com）：基于注意力、联想与记忆研究，融合心理学与行为经济学，提出 Smart Persuasion（智能说服）理论。
  • 同类竞品 Blog：FreshMarketer、VWO、Optimizely、Omniconvert、Convert、吆喝科技（热云数据）、火山引擎等。

三、能力矩阵（应会）：六大能力

能力定义
洞察能力能够敏锐发觉问题，判断问题（产品/数据）
解决能力能够快速设计出/检索到可能的解决方案
抽象能力能够抽象产品的问题类别/解决路径，善于总结规律
还原能力能够将问题还原为导致问题的各类因素（外因/人因）
组织能力能够画清路线图，组织人员协作，引导试验方向
教练能力能够教授各类角色掌握试验全流程的问题处理方法

1. 洞察能力：敏锐发现并判断问题

  • 入门案例：非有效信息占据空间过大、核心价值主张难以阅读、图片与文字冲突、版式老旧。
  • 进阶案例：特权 Icon 看上去不可点击——涉及雅各布定律（用户把大部分时间花在别人家产品上，因此期望你的产品与别人有相同的操作模式）。
  • 修炼方法：大量收集、大量对比 + 设计原则学习。不仅发现问题，也要多发现闪光点（如扫描全能王运用价格诱饵效应、获得感前置、紧迫性、稀缺性、拟真价值传递等）。

2. 解决能力：快速设计/检索解决方案

  • 案例：一个搜索框有多少种可能性？需考虑占位文案、搜索推荐、推荐内容样式，以及默认/焦点/输入/完成/异常五种状态。

  • 修炼方法：大量积累整理设计方案与最佳实践 + 适配验证。

3. 抽象能力：总结规律、抽象类别与路径

以"工具订阅模块"为例，可抽象出四个层次：

  • ① 权益包装：单个权益的价值认定与服务边界划分（如彩云天气以功能包权益 vs 墨迹天气以场景包权益）。
  • ② 品类划分：基于人群定位、需求划分的权益组合（SPU 设计），博弈用户价值与平台商业价值。
  • ③ 定价策略：定价基准规则与差异化策略（SKU 设计），如一刻相册、有道云笔记通过价格诱饵引导连续订阅。
  • ④ 视觉工程：通过视觉设计最大化用户价值感知，用人因心理学诱导转化。

  • 修炼方法：练习总结学会分类，练习推演学会分层。

4. 还原能力：还原问题的人因与外因

  • 还原人因：站在用户视角穷举决策顾虑（如报课时会想：能获得什么？质量是否有保障？是否适合我水平？会否很花时间？能否退费？是否支持在线练习？有无答疑？购买是否方便？当前有无优惠？）。
  • 还原外因：结合垂直领域认知深浅层还原（如中信证券 vs 浙商证券——定位、获客、核心客群、路线不同，导致要"打动专业投资者 · 要专业周全" vs "打动新手投资者 · 要简单降阻力"的差异）。

  • 修炼方法：充分收集用户决策因素 + 充分收集垂直领域认知。

5. 组织能力

  • 定义：画清路线图，组织人员协作，引导试验方向。
  • 训练方法：接受客户的复杂命题，训练复杂问题的拆解；画清达成路径，安排内外部人员分工协作。

6. 教练能力

  • 定义：教授各类角色掌握试验全流程的问题处理方法。
  • 训练方法：解决问题、消化吸收，积极输出、获取反馈、持续迭代。

结语

A/B 测试专家之路：

  • 完成工作，有一些快捷的方法；
  • 知识学习，有一些明确的路径；
  • 能力训练，却没有快速的捷径——需做大量收集、大量刻意练习。
